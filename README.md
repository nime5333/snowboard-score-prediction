# FIS Snowboard Big Air — Score Prediction Model
### Machine Learning Class Project

A machine learning model that predicts the judged score of a FIS Snowboard Big Air trick,
given its technical characteristics. Built using real competition data from 7 FIS World Cup
events across the 2024/25 and 2025/26 seasons.

---

## Project Goal

FIS Snowboard Big Air judging has no publicly standardized scoring formula. This model
attempts to quantify the relationship between trick difficulty and judge scores, providing
athletes and coaches a tool to estimate the potential value of a successfully landed trick
during training and competition preparation.

> **Note:** The model assumes medium-to-high execution quality. It is not designed to
> predict scores for falls or bails — only successfully landed attempts.

## Data

### Source
All data comes from official FIS result PDFs, publicly available at:
**https://www.fis-ski.com/DB/snowboard/park-and-pipe/calendar-results.html**

Each PDF contains one row per jump attempt including: athlete name, country, stance,
run number, TrickID (encoded trick), individual judge scores (J1–J6), jump score, and
jump ID (A or B).

### Events Included

| Competition | Location | Season |
|---|---|---|
| FIS World Cup Big Air | Chur, Switzerland | 2024/25 |
| FIS World Cup Big Air | Klagenfurt, Austria | 2024/25 |
| FIS World Cup Big Air | Kreischberg, Austria | 2024/25 |
| FIS World Cup Big Air | Beijing, China | 2024/25 |
| FIS World Cup Big Air | Aspen, USA | 2024/25 |
| FIS World Cup Big Air | Secret Garden, China | 2025/26 |
| FIS World Cup Big Air | Steamboat, USA | 2025/26 |

### Dataset Stats
- Raw rows: 905
- After filtering (65+ score cutoff): 343
- Unique athletes: 100
- Gender: Men's only

### Filtering Decision
Scores below 65 were excluded as they consistently reflect falls or major execution
failures rather than trick difficulty. This is consistent with the FIS Judges Handbook
Section 10 crash range (1–30 points) and Section 11.2 deduction scale. The model is
designed to predict the value of a landed trick, not the outcome of a fall.

## Features

All features are derived from the FIS TrickID column and competition metadata.

| Feature | Description | FIS Handbook Reference |
|---|---|---|
| SpinDegrees | Total rotation in degrees (e.g. 1800 = 1800°) | Section 8.2.1 — Amount of Rotation |
| Difficulty | Composite difficulty score (1–10) | Section 8.2 — Difficulty |
| Switch | 1 if switch/fakie takeoff (x- prefix or Cab) | Section 8.2.1 — Blind Landings |
| DoubleCork | 1 if double cork modifier present | Section 8.2.1 — Axis |
| TripleCork | 1 if triple cork modifier present | Section 8.2.1 — Axis |
| Rodeo | 1 if rodeo/off-axis modifier present | Section 8.2.1 — Axis |
| DirectionEnc | Rotation direction (0=frontside, 1=backside, 2=Cab) | Section 8.2.1 — Direction |
| StanceGoofy | 1 if athlete rides goofy (right foot forward) | Section 16 — Snowboard Dictionary |
| GrabDifficulty | Grab difficulty rating (1–5 scale) | Section 8.1.1 — Grabs |
| RunNum | Run attempt number (1, 2, or 3) | Section 9.1 — Anchor Scoring |
| JumpIDB | 1 if Jump B attempt | Section 9.1 — Anchor Scoring |

### TrickID Format
FIS encodes tricks as hyphen-separated codes, for example:

```
x - b  - D  - 12  - Ng
│   │    │    │     └── Grab: Nosegrab
│   │    │    └──────── Spin: 1260°
│   │    └───────────── Modifier: Double Cork
│   └────────────────── Direction: Backside
└────────────────────── Prefix: Switch takeoff
```

## Results

### Model Comparison (65+ score cutoff, 343 rows)

| Model | MAE | RMSE | R² | CV MAE |
|---|---|---|---|---|
| Linear Regression (Baseline) | 5.456 | 6.349 | 0.4883 | 5.771 |
| **Random Forest** | **5.065** | **5.942** | **0.5517** | **5.471** |
| XGBoost | 5.579 | 6.668 | 0.4356 | 5.990 |

**Best model: Random Forest** with MAE of 5.07 points and R² of 0.55.

For context, FIS judges themselves typically vary by 3–5 points per jump, meaning the
model's error is within the natural range of human judging variance.

### Key Finding
Rodeo was the most important feature despite being a modifier rather than a primary
spin measurement. This reflects a pattern found in the data and consistent with FIS Section 8.2.1.

## Known Limitations

- **Men's data only** — all 7 events are men's competitions. Women's scoring patterns
  may differ and would require separate data collection.
- **Execution not captured** — the TrickID encodes what trick was attempted, not how
  well it was landed. Progression, style, and amplitude are not represented.
- **~15 rows excluded** — trick codes like `T-Mo`, `bob`, and `McB` are non-standard
  and could not be reliably parsed into features.
- **Small dataset** — 343 training rows is sufficient for a class project but a
  production model would benefit from more seasons of data.
- **Men's World Cup only** — Olympic and continental cup events were not included.
